Digest Calculations

Digest Algorithm Input Data

Digests are computed from three components: * A pair of functions consisting of a one-way hash function H(d) and digest KD(s,d), where s stands for secret and d stands for data * A chunk of data containing security information, including the secret password, called A1 * A chunk of data containing nonsecret attributes of the request message, called A2 The two pieces of data, A1 and A2, are processed by H and KD to yield a digest.

H() = MD5() KD(,) = H(concatenate(:)) qop: quality of protection : HTTP request method : the request URI from the request line nc: nonce count
